Generally above issue is observed with oacore managed servers. In case, we receive an alert "java.lang.OutOfMemoryError" associated with an oacore managed server, please perform the steps given below in "fix" section. 


Fix:
===
1. Note down the oacore_server<XX> managed server mentioned in the above error message.

2. Login to WebLogic Admin console and check the "Health" status of that managed server.

Admin console URL:-

When fs1 is the run filesystem - Weblogic Admin Console: http://c7-a1-oa2-iprd-73.cisco.com:7052/console

When fs2 is the run filesystem - Weblogic Admin Console: http://c7-a1-oa2-iprd-73.cisco.com:7053/console

3. If the "Health" column against that oacore managed server is blank or showing warning status, login to the application node where that managed server is running as "oacfnprd" user, search for its process-id and kill it.

ps -ef|grep -i oacore_serverXX

kill -9 <pid returned in above command> 

4. As soon as the process id of the managed server is killed, node manager on that node will automatically start the managed server. Check oacore managed server's status from Admin console. Ensure that it comes to "Running" state.
