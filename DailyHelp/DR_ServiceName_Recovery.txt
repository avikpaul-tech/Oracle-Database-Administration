3. start the first or second instnace in nomount state.

startup nomount;

4. connect the rman prompt and create ctl file from primary using primary db_unique_name/service name.

rman
CONNECT TARGET "SYSBACKUP/test11g AS SYSBACKUP"
connect catalog rman_zdwhprd/XB_7r_h4Q0@rcatprod
RUN
{
ALLOCATE CHANNEL c1 DEVICE TYPE disk;
ALLOCATE CHANNEL c2 DEVICE TYPE disk;
RESTORE STANDBY CONTROLFILE FROM SERVICE ZDWHPRD;
}

5. place the db in mount state using new control file in rman prompt

ALTER DATABASE MOUNT;
exit

6. check the db status 

set lines 1000

select name, open_mode , FS_FAILOVER_STATUS, FS_FAILOVER_OBSERVER_PRESENT, DATABASE_ROLE from gv$database;
